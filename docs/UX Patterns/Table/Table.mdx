# Table

Tables present and organize data in a structured, readable format. Aquarium ships an opinionated `Table` built on Ant Design — these docs cover the **patterns** that get layered on top: the cell rendering vocabulary, empty/error/loading states, filters, and pagination.

## When to use

- Read-list pages where each row maps to a navigable record (data models, pipelines, audit logs)
- Configurable surfaces where users sort, filter, and act on rows
- Any place a structured list with multiple attributes per item is more scannable than cards

For dashboards, summary tiles, or single-attribute lists, prefer `Card`, `StatisticsCard`, or `List`.

## What's documented

- **Cell Types** — rendering vocabulary for individual cells (text, link, status, tag, actions). Mix and match per column.
- **Patterns** — table-level behavior (empty / error / loading states, filters, sticky columns, server pagination). The recommended filter pattern is the **Inline Toolbar**; see Modal Filters only when there are too many dimensions to fit inline.
- **Examples** — end-to-end real-world tables (Data Models, etc.) showing how cells + patterns compose.
