# Alert

### Overview

Alerts are persistent UI components used to communicate important information to users. They remain visible on the page until manually dismissed or programmatically cleared. Alerts may be used to inform users of relevant system or process updates that don't require immediate action, or to prompt them to acknowledge or resolve an issue before proceeding.

<Story of={AlertStories.StorybookNote} />

### Alert Types

Four alert types are supported, each with distinct visual styling to convey the message's intent and importance: success, info, warning, and error.

### Closable Alerts

Alerts can be closable or not, depending on whether users need to dismiss them.

### Alerts with Links

Alerts can include links within the message text. Links should be styled with text color and underlined to maintain consistency with the alert's visual design.
