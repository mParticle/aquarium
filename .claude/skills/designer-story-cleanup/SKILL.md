# Designer Story Cleanup

Designers (and the AI tools they run via `/implement-ticket`) often contribute
new `*.stories.tsx` files that "work" visually but bypass Aquarium primitives â€”
raw `<div style={{display:'flex',...}}>` instead of `<Flex>`, inline-style
overrides on `Typography.Text` instead of its props, and the occasional
copy-paste artifact like `'Primary\n'` in an arg. This skill catches and fixes
those before the maintainer has to.

## When to use

Run this skill before reviewing/merging any new or edited `*.stories.tsx` file
that was authored by a non-engineer or generated by `/implement-ticket`. Also
run it when explicitly asked to "clean up", "review", or "tidy" a story file.

**Not for**: production component source under `src/components/<X>/<X>.tsx`.
Those are reviewed against the full Aquarium contract, not this lighter
checklist.

## The checklist

Apply these rules in order. Each one is a hard fix, not a suggestion.

### 1. No raw `<div>` for layout

If a `<div>` exists only to apply `display: flex`, `flexDirection`, `gap`,
`justifyContent`, or `alignItems`, replace it with `<Flex>` and move the
remaining (visual) styles to `style={{ ... }}` on the `Flex`.

Mapping:

| Inline style               | `<Flex>` prop        |
| -------------------------- | -------------------- |
| `display: 'flex'`          | (implicit)           |
| `flexDirection: 'column'`  | `vertical`           |
| `gap: <token>`             | `gap={<token>}`      |
| `justifyContent: 'center'` | `justify="center"`   |
| `alignItems: 'center'`     | `align="center"`     |
| `alignItems: 'flex-start'` | `align="flex-start"` |

Keep `alignSelf`, `width`, `background`, `padding`, `border*`, `boxShadow`
inside `style={{}}` on the `Flex` â€” those aren't `Flex` props.

If a layout would just be `<Flex justify="center" align="center">`, prefer
`<Center>`.

### 2. Typography uses props, not inline styles

`Typography.Text` / `Typography.Title` / `Typography.Paragraph` /
`Typography.Link` accept these props directly â€” use them instead of
`style={{...}}` overrides:

| Inline style                      | Use prop                    |
| --------------------------------- | --------------------------- |
| `fontSize: FontSizeSm`            | `size="sm"`                 |
| `fontSize: FontSizeLg`            | `size="lg"`                 |
| `fontWeight: FontWeightStrong`    | `strong`                    |
| `fontStyle: 'italic'`             | `italic`                    |
| `textDecoration: 'underline'`     | `underline`                 |
| `textDecoration: 'line-through'`  | `delete`                    |
| `color: ColorTextTertiary` (etc.) | `color="ColorTextTertiary"` |

The `color` prop accepts the full set of `TypographyColor` keys (see
`src/components/general/Typography/colors.ts`). Passing the token name as a
string is correct â€” do **not** import the token and pass it via `style`.

Things that legitimately stay in `style`: `textTransform`, `margin: 0` (to
strip antd's default Title margin), and any `letterSpacing` / `lineHeight` not
already covered by `size`.

### 3. Design tokens, not raw values

Already enforced project-wide (see `CLAUDE.md`), but worth re-checking when
cleaning a designer's file:

- âś… `gap={MarginXs}` not `gap="8px"`
- âś… `padding: PaddingLg` not `padding: 24`
- âś… `borderRadius: BorderRadiusLg` not `borderRadius: 8`

After removing inline-style overrides per rule 2, prune the corresponding
imports from `'src/styles/style'` so the import block doesn't list dead tokens.

### 4. String-literal hygiene in story args

Designer-authored `args` blocks are a hotspot for copy-paste artifacts. Scan
every string literal in `args:` / `defaultArgs` and look for:

- Trailing `\n` or `\t` in labels (e.g. `'Primary\n'` â€” almost always a paste
  artifact, never intentional)
- Leading/trailing spaces (`' Submit'`, `'Submit '`)
- Smart quotes (`'donâ€™t'`) where straight quotes are expected
- Mixed casing in `eyebrow` / button labels that doesn't match the rest of the
  file's tone

These are silent bugs â€” they render but look wrong. Strip them.

### 5. Don't reinvent existing components

Before keeping a hand-rolled card / banner / pill / chip, check whether
`Card`, `Alert`, `Tag`, `Badge`, or `Statistic` (under `src/components/`)
already does it. If a "Custom X" story is intentionally demonstrating a
composition pattern (e.g. `CardCustomCards.stories.tsx`), that's fine â€” but
the _primitives_ inside the composition should still be Aquarium components,
not divs.

### 6. Imports

- Pull from `'src/components'`, not deep paths or `'antd'` directly.
- Drop unused imports after the rewrite (rules 1 and 2 typically free up
  several token imports).
- Keep `import React from 'react'` if Storybook's JSX runtime in this repo
  needs it â€” don't remove it speculatively.

## Verification

After applying fixes, in this order:

1. `npx tsc --noEmit` â€” must pass; the project's CI runs it and `npm run build`
   alone won't catch story-file type errors.
2. Eyeball the diff. The cleaned file should have **fewer** `style={{...}}`
   blocks and **fewer** token imports than before, not more.
3. Don't commit regenerated `src/styles/style.ts` or `src/styles/_variables.css`
   if the task didn't touch tokens (`git checkout --` them).

## Output

Make the edits directly. Then summarise what changed in one short block:

```
Replaced N raw <div>s with <Flex>
Moved M Typography style overrides to props
Fixed K string-literal artifacts in args
Pruned X unused token imports
```

Don't lecture the contributor â€” the skill exists so they don't have to learn
this themselves.
