# Common Component Modification Patterns

Reference for implementing typical design system changes in Aquarium.

## Adding a New Component Variant (e.g., new Alert type, new Button style)

### Pattern: CSS Override on Ant Design

When Ant Design doesn't support the variant natively, use this approach:

1. **Create a custom type union** extending Ant's types
2. **Map the custom type to the closest Ant type** under the hood
3. **Apply a CSS class** to override visual styles
4. **Add design tokens** for the new colors

### Files to modify (in order):

| Step | File                                                              | What to do                                                                                |
| ---- | ----------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| 1    | `design/GlobalToken.json`                                         | Add new color tokens (e.g., `colorMyVariant`, `colorMyVariantBg`, `colorMyVariantBorder`) |
| 2    | Run `npx style-dictionary build --config ./style-dictionary.json` | Regenerates `src/styles/style.ts` and `src/styles/_variables.css`                         |
| 3    | `src/components/<category>/<Component>/<Component>.tsx`           | Add variant to type union, map to Ant type, add CSS class                                 |
| 4    | `src/components/<category>/<Component>/<component>.css`           | Add CSS overrides using `var(--token-name)`                                               |
| 5    | `src/components/<category>/<Component>/<Component>.stories.tsx`   | Add stories for the new variant                                                           |
| 6    | `src/components/index.ts`                                         | Export any new types                                                                      |

### Example: Adding a "default" gray Alert type

```tsx
// Custom type extending Ant's built-in types
type AlertType = 'success' | 'info' | 'warning' | 'error' | 'default'

// In the component — map to closest Ant type + add CSS class
const isDefault = type === 'default'
const antType = isDefault ? 'info' : type
const defaultClass = isDefault ? 'ant-alert-default' : ''
```

```css
/* CSS overrides using design token CSS variables */
.ant-alert-default.ant-alert-info {
  background: var(--rokt-gray-1);
  border-color: var(--rokt-gray-4);
}
.ant-alert-default.ant-alert-info .ant-alert-icon {
  color: var(--rokt-gray-6);
}
```

## Adding Design Tokens

**IMPORTANT**: `src/styles/style.ts` and `src/styles/_variables.css` are auto-generated by style-dictionary. Never edit them directly.

1. Add tokens to `design/GlobalToken.json` in camelCase:
   ```json
   "colorMyToken": { "value": "#hex" }
   ```
2. Run: `npx style-dictionary build --config ./style-dictionary.json`
3. The token becomes available as:
   - TypeScript: `ColorMyToken` (PascalCase) in `src/styles/style.ts`
   - CSS: `var(--color-my-token)` (kebab-case) in `src/styles/_variables.css`

### Token naming conventions

Follow the existing pattern for semantic color tokens:

- `colorFoo` — the main color (e.g., icon color)
- `colorFooBg` — background color
- `colorFooBorder` — border color

Use values from the existing Rokt palette when possible (`RoktGray1`–`RoktGray10`, `RoktBrand1`–`RoktBrand10`).

## Adding Storybook Stories

Follow the existing pattern in the component's stories file:

```tsx
export const MyVariant: Story = {
  args: {
    type: 'myVariant',
    message: 'Descriptive message text.',
    showIcon: true,
    style: { marginBottom: 0, width: '860px' },
  },
}
```

Standard story variants to include for a new type:

- `<Variant>` — basic with icon
- `<Variant>WithoutIcon` — without icon
- `<Variant>WithCloseButton` — with close/dismiss
- `<Variant>WithLink` — with inline link

## Verification Checklist

Always run before committing:

1. `npm run build` — library compiles
2. `npx tsc --noEmit` — no type errors (catches stories too)
3. `npm start` — Storybook renders correctly (manual check)
